# Wave A2.4.0 — Unified Weekly CIO Report

## Objective

Create one governed Weekly CIO Report that consumes the A2.3.3 Executive
Decision Integration and cannot publish an independent portfolio posture,
execution status, or capital-deployment recommendation.

## Released baseline

- Platform version: `v1.3.0`
- Configuration baseline: `CB-002`
- Release: Unified Weekly CIO Reporting

## Authoritative inputs

- Executive Decision State A233
- Report Action Cards A233
- Report Conflicts A233
- Report Data Readiness A233
- Return Attribution Summary A232
- Attribution Coverage Summary A2311
- Production Certification

## Outputs

- Weekly CIO Report A240
- Weekly CIO Report Archive A240
- Weekly CIO Report Validation A240

## Governance rules

1. Portfolio posture must equal A2.3.3.
2. Execution status must equal A2.3.3.
3. Capital deployment is authorized only when execution status is EXECUTABLE.
4. Portfolio return is suppressed below 80% return-attribution coverage.
5. Current holdings and external opportunities remain separate.
6. A2.3.3 conflicts must reconcile to the Weekly Report.
7. Every row retains report, decision-run, platform-version and baseline lineage.
8. The A2.3.3 source validation must be passing.

## Public entry points

- `foRunWeeklyCioReportA240(options)`
- `foRunWeeklyCioReportValidationA240(reportId, decisionRunId)`
- `foRunWeeklyCioReportLatestValidationA240()`
- `foRunWeeklyCioReportSmokeTestA240()`

## Scheduling

A2.4.0 is registered in the additive engine registry but is not inserted into
the daily Autonomous CIO Orchestrator. Weekly scheduling should be introduced
only after report acceptance and session-aware scheduling controls are approved.
