## Registry Authority test coverage

Automated tests verify:

- exactly the three approved existing registries are adopted at bootstrap;
- metadata contains name, owner, version, authority, callable validation
  function, and governed registration status;
- discovery is case-insensitive and supports name and metadata criteria;
- discovery results are immutable and omit registry-owned business data;
- a valid registration is accepted and immediately discoverable;
- duplicate names and duplicate ownership are rejected case-insensitively;
- missing metadata and missing validation functions are detected;
- invalid registration status is detected;
- failed and throwing registry-owned validation functions are detected;
- rejected registration does not mutate the catalog; and
- the no-write smoke-test entry point passes for the initial catalog.

## Evidence boundary

This record proves source inspection, static repository validation, automated
unit and validation tests, lint, syntax validation, Apps Script source
inventory, Markdown lint, and whitespace validation in the Engineering Lab
repository checkout.

It does not claim a `clasp push`, Apps Script Lab execution, workbook mutation,
production deployment, production workbook validation, release certification,
merge, tag, or wave closure. Registry Authority has no workbook access or
write path, so no workbook fixture or operational data was required for these
repository gates. The additive `foRunRegistryAuthoritySmokeTest()` entry point
is available for a separately authorized Lab execution.
