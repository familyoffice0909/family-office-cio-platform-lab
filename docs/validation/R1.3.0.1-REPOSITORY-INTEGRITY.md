# Wave R1.3.0.1 — Repository Integrity

## Purpose

Reconcile repository metadata and automated integrity controls with the
released `v1.3.0` / `CB-002` Unified Weekly CIO Reporting baseline. This wave
does not create a platform release or change the production baseline.

## Reconciled files

- `Config.js`, `package.json`, and `package-lock.json` version metadata
- `README.md`, `CHANGELOG.md`, and the A2.4.0 release record
- Weekly CIO static regression tests
- Repository validation in `scripts/validate-platform.js`
- Engineering Lab CI in `.github/workflows/ci.yml`

## Known non-blocking findings

`npm audit` reports five moderate findings: the direct `@google/clasp`
development dependency and its `googleapis`, `googleapis-common`, `gaxios`, and
`uuid` dependency chain. They predate this wave and remain outside scope;
dependency versions and ranges are not broadly upgraded as part of this
reconciliation.

## Runtime and investment impact

No Apps Script runtime behavior or investment business logic changes. Scoring,
reporting, risk, valuation, recommendations, portfolio decisions, and production
workbooks are unchanged. No deployment or `clasp push` is part of this wave.

## Rollback

Revert the single R1.3.0.1 commit with `git revert <commit>`, then rerun the
validation commands above. Do not move the existing `v1.3.0` tag or rewrite
shared history.
