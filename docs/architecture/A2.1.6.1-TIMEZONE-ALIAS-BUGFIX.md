# A2.1.6.1 Timezone Alias Validation Bug Fix

Google Sheets may return `America/Nassau` for a workbook configured in the UI as
Eastern Time – Toronto. Exact string comparison against `America/Toronto`
therefore produced a false validation failure.

The platform now accepts a controlled allowlist of equivalent Eastern-time IANA
identifiers and normalizes them logically to `America/Toronto`.

Fixed-offset zones such as Panama, Bogota, and Cayman remain invalid.
